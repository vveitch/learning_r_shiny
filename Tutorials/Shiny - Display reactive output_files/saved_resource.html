                                          Garrett Mod Palash Goyal 5 months ago

                                          This is due to submitButton. When you add submitButton to an app, nothing in the app will update until the user clicks the submit button. So if you move the slider around the text should not update. But if you then click the submit button, the text should update.

                                          submitButton is just an easy way to make an app only react when your user specifically asks for it to (by clicking the submit button).

                                            I think that it could be a good idea to add a word or two about checkboxes/conditional statements to the tutorial. As in, where / how to put them.

                                            My example: I wanted to have a plot plotted out only if a checkbox was checked. As a newbie, the most intuitive way for me to do it was:

                                            if ("1" %in% input$checkGroup) {output$plot1 <- renderPlot(hist(rnorm(50,0,1)))}

                                            Shiny, naturally, gave me an error and told me to put this all in a reactive statement. So I did:

                                            reactive({if ("1" %in% input$checkGroup) {output$plot1 <- hist(rnorm(50,0,1))})

                                            output$plot1 <- renderPlot({if("1" %in% input$checkGroup) {hist(rnorm(50,0,1))}})

                                                                                                      Garrett Mod isomorphisms 9 months ago

                                                                                                      That's correct except for the `reactive` part. You don't need to call `reactive` inside of a `render*` function.

                                                                                                      Think of it this way, the `render*` functions assemble a "live" object to save to `output`. `reactive` assembles a "live" expression that does _not_ get saved to `output`. Lesson 6 explains when and why you would want to use `reactive`.
