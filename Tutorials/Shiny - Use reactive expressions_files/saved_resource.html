                                                        Garrett Mod Costas 6 months ago

                                                        Costas, Interesting question. I didn't realize that the action button was still in there. I originally planned for this lesson to be much longer, at the end you would build this server.R script, which delays pinging yahoo for a stock until you click the action button:


                                                        # server.R

                                                        library(shiny)
                                                        library(quantmod)
                                                        source("helpers.R")

                                                        shinyServer(function(input, output) {
                                                        dataInput <- reactive({
                                                        if(input$get == 0) return(NULL)
                                                        isolate({
                                                        getSymbols(input$symb, src = "yahoo",
                                                        from = input$dates[1], to = input$dates[2],
                                                        auto.assign = FALSE)

                                                        })

                                                        })

                                                        finalInput <- reactive({
                                                        if (!input$adjust) return(dataInput())
                                                        adjust(dataInput())
                                                        })

                                                        output$plot <- renderPlot({
                                                        if(input$get == 0) return(NULL)

                                                        chartSeries(finalInput(), theme = chartTheme("white"),
                                                        type = "line", log.scale = input$log, TA = NULL)
                                                        })

                                                        })

                                                        What you want to do is a little different. You want the dataInput reactive expression to run even if the only thing observing it (the plot) does not update. You are correct that Shiny will not execute a reactive expression unless an observer (the end of a chain) asks for the expression to be rerun. The solution is to create a second observer with observe that just keeps an eye on the reactive expression. This observer won't do anything except ensure that the expression stays up to date. I'm making it print the first value of dataImport(), just so you can see that dataImport() actually changed.

                                                        Notice how I use isolate in both of these scripts. Controlling reactions takes two things

                                                        1. use isolate to prevent automatic reactions. Objects that call a reactive value wrapped in isolate won't respond when the reactive value changes.

                                                        2. Call an action button that will trigger an observer (or expression) to update. When the expression updates, it will look up the current values of all of its input - even the input surrounded with isolate, which is how we end up with controlled "reactions."


                                                        # server.R

                                                        library(shiny)
                                                        library(quantmod)
                                                        source("helpers.R")

                                                        shinyServer(function(input, output) {
                                                        dataInput <- reactive({
                                                        getSymbols(input$symb, src = "yahoo",
                                                        from = input$dates[1], to = input$dates[2],
                                                        auto.assign = FALSE)
                                                        })

                                                        observe({
                                                        dataInput()
                                                        print(dataInput()[1,1])
                                                        })

                                                        finalInput <- reactive({
                                                        if (!input$adjust) return(dataInput())
                                                        adjust(isolate({dataInput()}))
                                                        })

                                                        output$plot <- renderPlot({
                                                        if(input$get == 0) return(NULL)

                                                        chartSeries(finalInput(), theme = chartTheme("white"),
                                                        type = "line", log.scale = input$log, TA = NULL)
                                                        })
                                                        })

                                                                      Garrett Mod Lthakali 6 months ago

                                                                      Lthakali, observe and reactive are similar: both take code that contains reactive values (e.g. widget values and other things that might change).

                                                                      However, reactive will return a value when it runs its code and observe will not return a value.

                                                                      So you would use observe to generate side effects. For example, you could use observe to source a file or write to a file whenever a value changes. These things are important, but they do not return a value for a second R function to use.

                                                                      You would use reactive to compute a new value (or build a new object) every time a reactive value changes. You would then have a second function use the new value. Since reactive creates a new value (and values are only ever used by other functions), we were able to program some time saving features into reactive. For example, Shiny won't actually run the code in a reactive expression until another function asks for it's value. If nothing uses the value, Shiny will effectively ignore the reactive expression.
